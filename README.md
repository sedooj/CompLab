## 10. Грамматика

### 10.1 Грамматика в нотации Хомского

```
1.   <START> -> 'val' <VAL>
2.   <VAL>   -> <SPACE>
3.   <SPACE> -> <IDENTIFIER> <COLON>
4.   <COLON> -> ':' <ARG_LIST>
5.   <ARG_LIST> -> '(' <ARG_LIST_TAIL>
6.   <ARG_LIST_TAIL> -> <TYPE> <ARG_LIST_TAIL_CONT>
7.   <ARG_LIST_TAIL_CONT> -> ',' <TYPE> <ARG_LIST_TAIL_CONT> | ')' '->' <ARROW_1>
8.   <ARROW_1> -> <TYPE> <ASSIGN>
9.   <ASSIGN> -> '=' <LBRACE>
10.  <LBRACE> -> '{' <PARAM_LIST> <ARROW_2>
11.  <PARAM> -> <IDENTIFIER> | '_'
12.  <PARAM_LIST> -> <PARAM> <PARAM_LIST_CONT>
13.  <PARAM_LIST_CONT> -> ',' <PARAM> <PARAM_LIST_CONT> | ε
14.  <ARROW_2> -> '->' <EXPR> <RBRACE>
15.  <EXPR> -> <TERM> <EXPR_TAIL>
16.  <EXPR_TAIL> -> '+' <TERM> <EXPR_TAIL> | '-' <TERM> <EXPR_TAIL> | ε
17.  <TERM> -> <FACTOR> <TERM_TAIL>
18.  <TERM_TAIL> -> '*' <FACTOR> <TERM_TAIL> | '/' <FACTOR> <TERM_TAIL> | ε
19.  <FACTOR> -> <IDENTIFIER> | <NUMBER> | '(' <EXPR> ')'
20.  <RBRACE> -> '}' <END>
21.  <END> -> ';'
22.  <TYPE> -> 'Int' | 'String' | 'Double' | 'Float' | 'Boolean'
23.  <IDENTIFIER> -> letter <ID_TAIL>
24.  <ID_TAIL> -> letter <ID_TAIL> | digit <ID_TAIL> | '_' <ID_TAIL> | ε
25.  <NUMBER> -> digit <NUM_TAIL>
26.  <NUM_TAIL> -> digit <NUM_TAIL> | ε
```

### 10.2 Грамматика в формате ANTLR (РБНФ)

```antlr
grammar FunctionDef;

// Parser rules (lowercase)
start
    : 'val' val_stmt
    ;

val_stmt
    : identifier ':' arg_list '=' lbrace
    ;

arg_list
    : '(' arg_list_tail
    ;

arg_list_tail
    : type (',' type)* ')' '->' type
    ;

lbrace
    : '{' param_list '->' expr '}'
    ;

param_list
    : identifier (',' identifier)*
    ;

expr
    : term (('+' | '-') term)*
    ;

term
    : factor (('*' | '/') factor)*
    ;

factor
    : identifier
    | number
    | '(' expr ')'
    ;

type
    : 'Int'
    | 'Float'
    | 'Double'
    | 'String'
    | 'Boolean'
    ;

identifier
    : IDENT
    ;

number
    : INT
    ;

// Lexer rules (uppercase)
IDENT
    : LETTER (LETTER | DIGIT | '_')*
    ;

INT
    : DIGIT+
    ;

LETTER
    : [a-zA-Z]
    ;

DIGIT
    : [0-9]
    ;

WS
    : [ \t\r\n]+ -> skip
    ;
```

## 11. Классификация грамматики

Разработанная грамматика классифицируется как **контекстно-свободная грамматика (КС-грамматика) типа LL(1)** по классификации Хомского.

**Обоснование:**

1. **Контекстно-свободная:** Каждое правило имеет вид $A \to \alpha$, где $A$ — нетерминал, $\alpha$ — цепочка терминалов и нетерминалов. Это соответствует определению контекстно-свободной грамматики.

2. **LL(1):** Грамматика может быть проанализирована алгоритмом рекурсивного спуска с предпросмотром одного символа (LL — Left-to-right, Leftmost derivation).

3. **Эквивалентна типу 2 в иерархии Хомского:** КС-грамматики расположены на уровне 2 иерархии Хомского (между регулярными и контекстно-зависимыми грамматиками).

## 12. Метод анализа

### 12.1 Рекурсивный спуск (Recursive Descent)

Реализован метод анализа **рекурсивный спуск** — позволяющий построить синтаксическое дерево путем рекурсивного вызова процедур, соответствующих нетерминалам грамматики.

**Алгоритм:**

1. Создается процедура для каждого нетерминала
2. Процедура проверяет соответствие текущего токена ожидаемому
3. Если токен совпадает, производится переход к следующему
4. Если нет — генерируется ошибка и запускается процедура восстановления
5. Процедуры вызываются рекурсивно согласно правилам грамматики

**Класс:** `SyntaxAnalyzer` в файле `syntax_analyzer.py`

**Основные методы:**

```python
def analyze(self, lexemes: List[Lexeme]) -> ParseResult:
    """Основная точка входа для синтаксического анализа"""

def _parse_start(self) -> None:
    """Парсит начальную конструкцию <START> -> 'val' <VAL>"""

def _parse_expr(self) -> None:
    """Парсит арифметическое выражение"""

def _match(self, expected_code: int) -> Lexeme:
    """Пытается сопоставить ожидаемый код токена"""

def _synchronize(self, sync_tokens: List[int]) -> None:
    """Синхронизация при ошибке (метод Айронса)"""
```

## 13. Диагностика и нейтрализация синтаксических ошибок (Метод Айронса)

### 13.1 Обнаружение ошибок

Синтаксический анализатор обнаруживает следующие типы ошибок:

1. **Пропущенный токен:** например, отсутствие `:` после идентификатора
2. **Неожиданный токен:** например, неправильный оператор
3. **Неправильный тип данных:** например, вместо типа указан идентификатор
4. **Незавершенное выражение:** например, отсутствие `;` в конце
5. **Недопустимые символы:** обнаруживаются еще на ЛА, передаются в СА

### 13.2 Нейтрализация ошибок (Метод Айронса)

**Метод Айронса** реализован через **синхронизацию (synchronization)**:

```python
def _synchronize(self, sync_tokens: List[int]) -> None:
    """Airons' error recovery: skip tokens until we find a synchronization point"""
    while self.current_pos < len(self.lexemes):
        if self._peek_code() in sync_tokens:
            break
        self._advance()
```

**Алгоритм:**

1. При обнаружении ошибки парсер сохраняет описание ошибки
2. Парсер пропускает токены до ближайшей точки синхронизации
3. Точками синхронизации служат ключевые токены: `;`, `}`, `)`
4. Парсер продолжает анализ, пытаясь обнаружить следующие ошибки
5. Этот подход позволяет найти несколько ошибок за один проход

**Пример:**

```
Входные токены: val f (Int) -> Int = { x -> x + 1 };
Ошибка #1:      ^^^^^^^^^ 
Сообщение:      Ожидалось ':'
Синхронизация:  пропуск до '('
Продолжение:    успешный парс остальной части
```

### 17.2 Класс SyntaxError

Структура для представления синтаксической ошибки:

```python
@dataclass
class SyntaxError:
    code: int = -1
    error_type: str = "синтаксическая ошибка"
    unexpected_lexeme: str = ""
    expected: str = ""
    line: int = 1
    column_start: int = 1
    column_end: int = 1
    message: str = ""
```

### 17.3 Класс ParseResult

Результат синтаксического анализа:

```python
@dataclass
class ParseResult:
    success: bool = True
    errors: List[SyntaxError] = field(default_factory=list)
    recovered_errors: int = 0
    tree_nodes: List[str] = field(default_factory=list)
```
