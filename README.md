# Компилятор — Языковой процессор

## Название и цель лабораторной работы

**Название:** Лабораторная работа 1. Разработка пользовательского интерфейса (GUI) для языкового процессора

**Цель:** Разработка кроссплатформенного GUI-приложения с функциональностью анализа и обработки текстового кода. Приложение должно предоставлять удобный интерфейс для редактирования кода с поддержкой вкладок, подсветкой синтаксиса, нумерацией строк, обработкой ошибок и интернационализацией.

## Описание проекта

Приложение **Компилятор** — это полнофункциональный текстовый редактор с элементами IDE для работы с учебным языком программирования (Pascal-подобный синтаксис). Основные возможности:

- **Многовкладочное редактирование**: работа с несколькими файлами одновременно в отдельных вкладках
- **Подсветка синтаксиса**: автоматическое выделение ключевых слов, строк, комментариев, чисел и операторов
- **Нумерация строк**: отображение номеров строк с синхронизацией при прокрутке
- **Управление файлами**: создание, открытие, сохранение файлов с поддержкой Drag & Drop
- **Таблица ошибок**: вывод синтаксических, лексических и семантических ошибок с возможностью перехода к месту ошибки
- **Интернационализация**: поддержка русского и английского языков с переключением без перезагрузки
- **Масштабирование шрифта**: увеличение/уменьшение размера текста в редакторе и области результатов
- **Строка состояния**: отображение текущей позиции курсора, количества строк и флага изменений

## Инструкция по сборке и запуску

- pip (менеджер пакетов Python)
- 100 MB свободного места на диске

```bash
pip install PyQt6
```

Или из файла requirements (если существует):

```bash

python3 main.py
```

#### На Windows:
```bash
cd C:\path\to\CompLab1
python main.py
```

Для упаковки приложения в один исполняемый файл используйте `PyInstaller`:

1. Установите PyInstaller:
```bash
pip install pyinstaller
```

2. Соберите приложение:
```bash
pyinstaller --onefile --windowed --name="Compiler" --icon=images/favicon.ico main.py
```

3. Готовый файл `Compiler.exe` (Windows) или `Compiler` (Linux/macOS) будет находиться в папке `dist/`

## Описание интерфейса и функций (руководство пользователя)

### Открытие приложения

При запуске приложение отображает главное окно с пустой областью редактирования и сообщением:
```
Создайте новую вкладку (Ctrl+N)
или откройте файл (Ctrl+O)
```

### Главное меню

Приложение содержит следующие меню верхнего уровня:

#### 1. **Меню «Файл»** (File)

| Команда | Горячая клавиша | Описание |
|---------|-----------------|---------|
| Создать | `Ctrl+N` | Создаёт новую пустую вкладку |
| Открыть | `Ctrl+O` | Открывает диалог выбора файла и загружает его в новую вкладку |
| Сохранить | `Ctrl+S` | Сохраняет текущую вкладку; если файл новый, открывает диалог «Сохранить как» |
| Сохранить как… | `Ctrl+Shift+S` | Сохраняет текущую вкладку под новым именем |
| Выход | `Ctrl+Q` | Закрывает приложение; если есть несохранённые изменения, запросит подтверждение |

#### 2. **Меню «Правка»** (Edit)

| Команда | Горячая клавиша | Описание |
|---------|-----------------|---------|
| Отменить | `Ctrl+Z` | Отменяет последнее действие |
| Повторить | `Ctrl+Y` | Повторяет отменённое действие |
| Вырезать | `Ctrl+X` | Вырезает выделенный текст в буфер обмена |
| Копировать | `Ctrl+C` | Копирует выделенный текст |
| Вставить | `Ctrl+V` | Вставляет текст из буфера обмена |
| Удалить | `Del` | Удаляет выделенный текст |
| Выделить все | `Ctrl+A` | Выделяет весь текст в текущей вкладке |
| Увеличить текст | `Ctrl++` | Увеличивает размер шрифта на 1pt |
| Уменьшить текст | `Ctrl+-` | Уменьшает размер шрифта на 1pt |
| Сбросить масштаб | `Ctrl+0` | Восстанавливает размер шрифта по умолчанию (12pt) |
| Поиск | `Ctrl+F` | Заглушка поиска (зарезервировано для будущих версий) |

#### 3. **Меню «Текст»** (Text)

| Команда | Описание |
|---------|---------|
| Постановка задачи | Справочная информация о задании лабораторной работы |
| Грамматика | Описание используемой грамматики языка |
| Классификация грамматики | Классификация по типам (КБ, НБ и т.д.) |
| Метод анализа | Описaние метода синтаксического анализа |
| Тестовый пример | Пример кода для тестирования |
| Список литературы | Ссылки на информационные источники |
| Исходный код программы | Содержимое текущего файла в отдельном окне |

#### 4. **Меню «Пуск»** (Run)

| Команда | Горячая клавиша | Описание |
|---------|-----------------|---------|
| Пуск | `Ctrl+R` или `F5` | Запускает анализатор: парсит код в текущей вкладке и выводит результат в таблицу ошибок и область вывода |

#### 5. **Меню «Язык»** (Language)

| Команда | Описание |
|---------|---------|
| Русский | Переключает интерфейс на русский язык |
| English | Переключает интерфейс на английский язык |

#### 6. **Меню «Справка»** (Help)

| Команда | Горячая клавиша | Описание |
|---------|-----------------|---------|
| Вызов справки | `F1` | Открывает диалоговое окно со справкой по использованию |
| О программе | — | Показывает информацию о версии и авторе приложения |

### Панель инструментов

Панель расположена прямо под меню и содержит кнопки для быстрого доступа к основным функциям (с иконками):

- **Создать файл** — новая вкладка
- **Открыть файл** — диалог открытия
- **Сохранить** — сохранение текущей вкладки
- **Отменить/Повторить** — навигация по истории
- **Копировать/Вырезать/Вставить** — буфер обмена
- **Пуск** — выполнение анализатора
- **Размер шрифта** — выпадающее меню/спинбокс с выбором размера (6–72pt)
- **Справка/О программе** — быстрый доступ к информации

### Область редактирования (левая часть, 2/3 ширины)

#### Вкладки редактора
- Каждый открытый файл отображается в отдельной вкладке
- Название вкладки показывает имя файла; если текст изменён, добавляется звёздочка `*`
- Двойной клик по пустой области таб-бара создаёт новую вкладку
- Каждую вкладку можно закрыть кнопкой `×` на самой вкладке

#### Текстовый редактор (CodeEditor)
- **Нумерация строк**: слева отображаются номера строк, синхронизированные с прокруткой
- **Подсветка синтаксиса**:
  - Ключевые слова (program, begin, end, if, while и т.д.) — **жирный оранжевый** цвет
  - Числа (123, 45.67) — синий цвет
  - Строки ('текст' или "текст") — зелёный цвет
  - Комментарии (// или { }, (* *)) — серый курсив
  - Операторы (:=, <=, >=, <>, +, -, *, /) — коричневый цвет
- **Горячие клавиши редактирования**: Ctrl+Z, Ctrl+Y, Ctrl+X, Ctrl+C, Ctrl+V, Del

#### Drag & Drop
- Можно перетащить один или несколько файлов из файлового менеджера прямо в редактор
- Каждый файл откроется в отдельной новой вкладке

---

### Область результатов (правая часть, 1/3 ширины)

#### Вкладки результатов
Область разделена на три фиксированные вкладки:

##### **Вкладка «Ошибки»**
Таблица с колонками:
- **Строка** — номер строки, где найдена ошибка
- **Столбец** — позиция символа в строке
- **Тип ошибки** — категория (Лексическая, Синтаксическая, Семантическая и т.д.)
- **Описание** — текст ошибки

Функции:
- Таблица только для чтения
- Двойной клик на строке автоматически переводит курсор в редакторе на указанную строку и столбец
- При нажатии кнопки «Пуск» таблица заполняется результатами анализа

##### **Вкладка «Вывод»**
- Текстовое поле только для чтения
- Отображает обычный текстовый вывод анализатора (количество строк, символов, выполненные операции)

##### **Вкладка «Лог»**
- Текстовое поле только для чтения
- Отображает отладочную информацию и сообщения анализатора
- Полезна для диагностики и отслеживания работы программы

### Строка состояния (внизу окна)

Отображает в реальном времени информацию о текущем состоянии:

| Элемент | Пример | Описание |
|---------|--------|---------|
| **Позиция курсора (строка)** | `Стр: 5` | Текущая строка |
| **Позиция курсора (столбец)** | `Кол: 12` | Текущий столбец в строке |
| **Количество строк** | `Строк: 42` | Общее количество строк в документе |
| **Флаг изменений** | `Изменён` | Показывается только если в текущей вкладке есть несохранённые изменения |
| **Размер шрифта** | `Шрифт: 12pt` | Текущий размер шрифта |

### Практические примеры использования

#### Пример 1: Создание и сохранение файла

1. Запустите приложение
2. Нажмите `Ctrl+N` (или кнопка «Создать» на панели) → появляется новая вкладка
3. Введите код:
   ```pascal
   program Hello;
   begin
     writeln('Привет, мир!');
   end.
   ```
4. Нажмите `Ctrl+S` → откроется диалог выбора папки и имени файла
5. Введите имя `hello.pas` и сохраните
6. Вкладка переименуется в `hello.pas`, звёздочка исчезнет

#### Пример 2: Открытие файла через Drag & Drop

1. Откройте файловый менеджер
2. Выберите файл `test.txt`
3. Перетащите его в окно приложения
4. Файл автоматически откроется в новой вкладке с подсветкой синтаксиса

#### Пример 3: Анализ кода и исправление ошибок

1. Откройте файл с кодом (или введите новый)
2. Нажмите кнопку «Пуск» (или `Ctrl+R`)
3. В таблице вкладки «Ошибки» появятся демонстрационные ошибки:
   - Строка 1, Столбец 1: Лексическая — "Неопознанный символ '@'"
   - Строка 3, Столбец 10: Синтаксическая — "Ожидался ';' после выражения"
   - Строка 5, Столбец 5: Семантическая — "Переменная 'x' не объявлена"
4. Двойной клик на первой ошибке → курсор прыгнет на строку 1, столбец 1

#### Пример 4: Изменение масштаба

1. Выберите из меню **Правка → Увеличить текст** (или `Ctrl++`)
2. Текст в редакторе увеличится на 1pt (например, с 12pt до 13pt)
3. Размер обновится везде: редактор, таблица ошибок, область вывода
4. Строка состояния покажет новый размер: `Шрифт: 13pt`
5. Нажмите `Ctrl+0` → размер сбросится на 12pt

#### Пример 5: Переключение языка

1. Откройте меню **Язык**
2. Выберите **English** → весь интерфейс переключится на английский (меню, кнопки, диалоги, строка состояния)
3. Выберите **Русский** → вернуться на русский язык

## Ограничения

### Известные ограничения реализации

1. **Анализатор — заглушка**
   - Функция «Пуск» выводит демонстрационные ошибки, а не реальный анализ кода
   - Полная реализация парсера языка требуется в следующих версиях

2. **Поиск и замена — не реализован**
   - Горячая клавиша `Ctrl+F` зарезервирована, но функционал отсутствует

3. **Подсветка синтаксиса**
   - Реализована для Pascal-подобного синтаксиса
   - Поддержка других языков требует создания новых классов наследников `QSyntaxHighlighter`

4. **Производительность**
   - При работе с файлами размером > 10 MB может наблюдаться замедление подсветки синтаксиса
   - Рекомендуется использовать файлы размером до 5 MB

5. **Откат кодировки**
   - Файлы сохраняются только в кодировке UTF-8
   - Поддержка других кодировок (CP1251, ANSI и т.д.) требуется в будущих версиях

6. **Многопроцессность**
   - Анализ кода выполняется в основном потоке
   - Для больших файлов рекомендуется использоватьQThread в будущих версиях

7. **История отката**
   - Рассчитана только на текущий сеанс
   - История теряется при закрытии вкладки

### Технические особенности

- Приложение использует модульную архитектуру с классами:
  - `CodeEditor` — редактор с нумерацией и подсветкой
  - `PascalHighlighter` — подсветка синтаксиса
  - `ResultTabWidget` — управление вкладками результатов
  - `CompilerWindow` — главное окно приложения

- Интернационализация реализована через словари переводов (в будущем можно перейти на QTranslator + .qm файлы)

- Drag & Drop перехватывается на уровне `CodeEditor`, а не главного окна

```pascal
program TestProgram;
```

### Расширение функциональности

Легко расширить приложение благодаря модульной архитектуре:

- **Добавить новый язык программирования**: создайте подкласс `PascalHighlighter`
- **Добавить новые горячие клавиши**: добавьте `QAction` в `_create_actions()` и подключите сигнал
- **Реализовать поиск**: создайте новый класс `FindDialog` и интегрируйте его в `CompilerWindow`
- **Добавить реальный парсер**: замените `on_run()` на вызов реального анализатора
